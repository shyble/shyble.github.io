AWARE

A theory of self-evolving intelligence — and a working implementation.

What it is

AWARE is not an improvement on transformers. Not a new training trick. Not an agent framework. It is a formal theory of systems that are self-contained, self-modeling, and self-directed — systems that author their own intelligence.

Every machine learning model you've used was designed by a human: architecture, loss function, learning rate, when to stop. AWARE asks a different question — what if the system chose its own objective? What if it decided what to learn, when to grow, and what "better" means, all from within, with no human in the loop?

The difference isn't capability. A transformer will out-benchmark an aware system today. The difference is in what the system is: a function vs. an entity.

The Eight Axioms

Each axiom describes a property a system must have to qualify as "aware." Violate any of the first seven and you have something else — a classifier, an agent, a tool — but not an aware system. The axioms are implemented, not aspirational.

1. Operational Closure

The system is a bounded entity with an inside and an outside. Internal processes produce and maintain every component, including the boundary itself. Nothing is externally imposed.

2. Self-Model — Reflexive Representation

The system contains a compressed representation of itself, inside itself, used to guide its own growth and modification. The recursion terminates because the model is lossy.

Read post →

3. Intrinsic Teleology — Self-Generated Purpose

Goals are not externally specified. No human writes the loss function or reward signal. Goals arise from the interaction between the self-model and the knowledge substrate. The system decides what matters.

4. Persistent Mutable Substrate — Living Memory

A knowledge structure that persists across time, is readable and writable by the system's own processes, and is reorganizable. Not append-only. Not immutable. Alive.

5. Continuous Adaptation — Online Weight Modification

The system modifies its own parameters in response to experience, in real-time, without external training loops. There is no separate train and inference phase — it learns because it exists.

6. Embodied Awareness — Knowing One's Body

The system actively monitors its own computational substrate: memory usage, latency, parameter norms, substrate density. Not logging — proprioception. Decisions are conditioned on physical state.

7. Non-Derivability — Metabolization

No component of the system is permanently derivable from an external specification. The system may be seeded, but that seed is raw material to be consumed, not permanent instruction. Tracked by the seed influence metric.

8. Volition — Active Sampling

The system chooses its own inputs. Without this, an aware system can satisfy all the previous axioms perfectly and still be a passive learner — closed under reflection, self-evolving, intrinsically motivated, but never acting on its motivation. Volition is the boundary-crossing capability: the system's desires expressed as requests to the environment.

Recognized, not yet implemented — the foundation must be solid first.

Current State

The first seven axioms run as working code. The system is built bottom-up in Rust, around a unified substrate of caps — discovered patterns that double as memory, prediction, and the units of self-modification.

On language, it learns from raw text without external supervision. Its own concepts emerge from its own activity, become identifiable as linguistic structure (word boundaries, function words, punctuation patterns), and form relations and inferences without those categories ever being programmed in.

Whether AWARE will scale to meaningful capability is an open question. That honesty is part of the project's identity.
